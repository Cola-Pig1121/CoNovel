# Anti-AI Edit Skill - 去AI味编辑技能

## 触发条件
当Agent需要检测和消除AI写作痕迹时加载此技能。

## 技能内容

### AI写作痕迹检测清单

#### 1. 高频AI词汇（必须替换）

| 类型 | AI词汇 | 替换建议 |
|------|--------|----------|
| 情感词 | 不禁 | 忍不住、情不自禁、下意识 |
| 比喻词 | 仿佛、宛如、恍若 | 像是、好像、如同 |
| 视觉词 | 映入眼帘 | 看见、望见、落入眼中 |
| 程度词 | 前所未有 | 破天荒、头一遭、从未有过 |
| 程度词 | 意义深远 | 重要、关键、影响深远 |
| 程度词 | 深刻影响 | 重大改变、带来转变 |
| 副词 | 微微 | 略微、稍许、轻轻 |
| 副词 | 淡淡 | 隐约、依稀、浅浅 |
| 副词 | 缓缓 | 慢慢、徐徐、渐渐 |
| 副词 | 默默 | 悄悄、无声地、暗自 |
| 结尾词 | 不由得 | 忍不住、情不自禁 |
| 结尾词 | 油然而生 | 涌上心头、自然而然 |

#### 2. AI句式模式

**三段式排比（需要打破）：**
```
不好：
他感到愤怒，感到悲伤，感到绝望。

好：
愤怒像火焰一样在胸口燃烧，几乎要将他吞噬。
```

**过度解释（需要删减）：**
```
不好：
他感到愤怒，这是因为他的朋友背叛了他，这种背叛让他感到非常痛苦。

好：
他握紧拳头，指节发白。
```

**学术论文式（需要重写）：**
```
不好：
在这种情况下，他做出了一个重要的决定，这个决定将对他的人生产生深远的影响。

好：
他深吸一口气，转身走向那扇门。
```

**正式语体入侵（需要口语化）：**
```
不好：
他对此表示赞同，并决定付诸行动。

好：
"行。"他点头，"干。"
```

#### 3. AI结构特征

**开头模式化：**
```
不好：
在这个风和日丽的早晨...

好：
阳光刺眼。
```

**结尾升华：**
```
不好：
这一刻，他明白了一个深刻的道理...

好：
他没有说话，只是握紧了手中的剑。
```

**过渡生硬：**
```
不好：
就在这时，一个人走了进来。

好：
门开了。
```

### 检测流程

#### Pass 1: 词汇扫描
逐段扫描，标记AI高频词汇：
```
[DETECT] 第15段: "不禁" -> 建议替换为"忍不住"
[DETECT] 第23段: "仿佛" -> 建议替换为"像是"
```

#### Pass 2: 句式分析
分析句式模式，识别AI特征：
```
[PATTERN] 第8段: 连续3个排比句 -> 建议打破
[PATTERN] 第12段: 学术论文式段落 -> 建议重写
```

#### Pass 3: 最小修正
只修改有问题的部分，保持原文风格：
```
[EDIT] 第15段: "他不禁微微一笑" -> "他忍不住轻轻一笑"
[KEEP] 第16段: 保持原文
```

### 修正原则

1. **最小修改** - 只改有问题的地方，不要大面积重写
2. **保持风格** - 修正后的文本要保持原文的风格和意图
3. **自然流畅** - 修正后的文本要读起来自然
4. **避免过度** - 不要把所有"仿佛"都改成"像是"
5. **保留特色** - 有些重复是作者有意为之，不要盲目删除

### 注意事项
1. AI痕迹检测要全面，但修正要克制
2. 修正后的文本要保持原文的风格和意图
3. 不要为了去AI味而破坏原文的美感
4. 有些"AI词汇"在特定语境下是合适的，不要一刀切
